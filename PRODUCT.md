## Design Context

### Users

Developer Mac owners — Homebrew, Docker, Node.js, Xcode users. They accumulate massive junk (node_modules, build artifacts, simulator caches, Docker images). They open Gargantua between coding sessions, not during. They trust open source, hate subscriptions, and are comfortable with CLI but want a polished GUI for maintenance tasks they don't want to think about. Secondary: privacy-conscious prosumers who care about telemetry.

### Brand Personality

Precise. Trustworthy. Gravitational.

Named after the black hole in Interstellar — this app devours your Mac's junk. The name isn't decorative; it shapes the entire visual language.

### Aesthetic Direction

Dark mode primary. Space-cold terminal that grew up — cold like deep space, precise like a file path, dense like a trading floor but not chaotic. Safety signals (green/amber/red from the Trust Layer) are warm and clear against the void.

**Theme:** Dark — developers in their working environment, often later in the day.
**Depth:** Borders only. No shadows. In space there are no point-light sources.
**Signature element:** Confidence orbit ring — a thin circular arc showing scan confidence %, inspired by Gargantua's orbital rings.

**Anti-references:** CleanMyMac X marketing polish. Generic dashboard templates. Glassmorphism. Neon-on-dark "AI aesthetic."

### Design Principles

1. **Trust is the product** — Every element serves the Trust Layer. If a user hesitates before clicking "Clean," the design failed.
2. **Classification over decoration** — Color means safety level. Accents mean interactivity. Nothing is decorative.
3. **Density with hierarchy** — Show a lot, but make the important thing obvious. File paths, sizes, confidence — all visible, all ranked.
4. **Terminal precision, GUI polish** — Monospace for data. System fonts for UI. Tabular numbers for alignment. No playfulness.
5. **Gravity, not glow** — Dark surfaces pull attention inward. Light elements float above them through elevation, not illumination.
