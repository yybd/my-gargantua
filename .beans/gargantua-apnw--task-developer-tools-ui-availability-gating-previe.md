**Key decisions:**
- View owns a three-case phase: `.loading`, `.ready(availabilities, previews)`, `.empty(availabilities)`. Per-tool preview state is `.loading | .loaded | .failed(String)` — the panel can show one tool's error without hiding a sibling tool's successful preview.
- Injected availability/preview providers via `@Sendable` closures so tests can derive state without running brew/docker subprocesses; default closures wrap `DeveloperToolPreviewAdapter()`.
- Pure reducers `deriveInitialPhase` and `applyPreviewResult` are static and internal so state transitions are covered by Swift Testing without SwiftUI rendering.
- Empty case carries the uninstalled availabilities so the UI can optionally show "not found" reasons per tool; avoids recomputing reasons in the view.
- `.task(id:)` runs the detached load; Task.isCancelled is checked before each preview to prevent stale completions bleeding into the next mount.
- Retry uses an unstructured Task kicked off by the button — accepted as a small leak bounded by the 15s preview timeout.
- Error logs mark `errorDescription` privacy as `.private` since brew/docker stderr can contain local paths and daemon URLs.
